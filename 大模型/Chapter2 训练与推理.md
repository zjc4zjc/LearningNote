# 1. 导读

首先，用大量文本做 **Pre-Training（预训练）**，就像让学生先把语文、数学基础都学一遍，具备基本“会答题”的能力。

接着做 **SFT（监督微调）**，相当于老师给标准答案让学生模仿练习，让模型学会“像人一样回答问题”，但这时候只是学会了“能答”，不一定“答得好”。

然后进入 **RLHF（人类反馈强化学习）** 的核心步骤：先收集一批同一个问题的多个回答，让人类去比较“哪个更好”，就像老师看两份试卷说“这份更优秀”。

|维度|SFT|RLHF|
|---|---|---|
|学习方式|监督学习|强化学习|
|训练目标|拟合标准答案|优化人类偏好|
|数据形式|(输入, 标准输出)|偏好排序（A 比 B 好）|
|是否涉及 reward|❌|✅|
|作用阶段|初始能力学习|行为对齐（alignment）|

## 3. SFT 有监督学习

SFT 和预训练在“训练形式上是一样的（都是下一词预测 + 交叉熵 loss）”，但“数据分布和目标不同”。

预训练的数据是无结构的自然连贯数据，来源于网页，书籍，代码等。对预测的所有token都算loss。

但SFT的数据是人工整理标签，有结构的，比如:`[用户问题] 如何写论文? [模型回答] 可以从引言开始...`。只对回答部分的token计算loss，不计算提问部分的loss

# 4. RLHF 人类反馈强化学习

## 4.1 大模型训练时涉及RL模型

大模型训练涉及的模型一共有四个主要模型，分别是：
- Actor Model：演员模型，这就是我们想要训练的目标语言模型
- Reward Model：奖励模型，它的作用是计算实际收益
- Critic Model：评论家模型，它的作用是预期收益
- Reference Model：参考模型，它的作用是给语言模型增加一些约束，防止语言模型训歪，使模型的回答结果最好与之前SFT模型的回答分布相近

基于人类偏好，训练一个 **Reward Model（奖励模型）**，它相当于变成一个“打分老师”，以后不用真人，每个回答都能自动给出一个“好不好”的分数。

接下来，用强化学习（通常是 PPO）去优化模型：**Actor Model（演员模型）** 不断生成回答，Reward Model 给它打分，同时 **Critic Model（评论家模型）** 提供“正常水平”的基准，然后模型根据“这次有没有比预期更好”来调整自己，让高分回答更容易被生成。

在这个过程中，还会加上 **Clip（限制更新幅度）** 防止模型超常发挥找到野路子，从而学偏，以及用 **Reference 模型（通常是 SFT 模型）做约束**，避免模型为了拿高分而变得不自然甚至胡说八道。
