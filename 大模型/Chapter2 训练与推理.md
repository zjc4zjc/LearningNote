# 1. 导读

首先，用大量文本做 **Pre-Training（预训练）**，就像让学生先把语文、数学基础都学一遍，具备基本“会答题”的能力。

接着做 **SFT（Supervised Fine-Tuning 监督微调）**，相当于老师给标准答案让学生模仿练习，让模型学会“像人一样回答问题”，但这时候只是学会了“能答”，不一定“答得好”。

然后进入 **Alignment (对齐阶段)**，主要分为两种方法：一是**RLHF（Reinforcement Learning from Human Feedback 人类反馈强化学习，代表有PPO/GRPO/DAPO/GSPO）** ，用强化学习做对齐。二是**Direct Preference Optimization（DPO）**，绕开强化学习直接用偏好做对齐。

| 维度          | SFT               | RLHF类                        | DPO类                 |
| ----------- | ----------------- | ---------------------------- | -------------------- |
| 学习方式        | 监督学习              | 强化学习                         | 监督学习                 |
| 训练目标        | 最大化数据似然<br>（拟合分布） | 最大化 reward<br>（人类偏好）         | 最大化偏好概率<br>（yc > yr） |
| 数据形式        | (输入, 标准输出)        | (输入, 标准输出)+<br>偏好排序（A 比 B 好） | (输入, 接受输出，拒绝输出)      |
| 是否涉及 reward | ❌                 | ✅                            | ❌                    |
| 作用阶段        | 基础能力 + 初步对齐       | 精细对齐<br>（alignment）          | 精细对齐<br>（alignment）  |
```
【对齐技术总览】
    │
    ├─ 1. RLHF 路线 (需要奖励模型 + RL算法)
    │      │
    │      ├─ PPO —— 经典RL算法，需要Critic模型
    │      │
    │      └─ GRPO —— PPO的变体，去掉Critic，用组内对比
    │            │
    │            ├─ DAPO —— GRPO的改进版 (针对长链推理)
    │            │
    │            └─ GSPO —— GRPO的另一改进版 (优化组采样策略)
    │
    └─ 2. 非RL路线 (直接偏好优化)
           │
           └─ DPO —— 用分类损失替代RL，无需奖励模型
```

# 3. SFT 监督微调

SFT 和预训练在“训练形式上是一样的（都是下一词预测 + 交叉熵 loss的监督学习）”，但SFT的数据是人工整理标签，有结构的，比如:`[用户问题] 如何写论文? [模型回答] 可以从引言开始...`。且只对回答部分的token计算loss，不计算提问部分的loss。

# 4. RLHF 人类反馈强化学习

## 4.1 PPO 近端策略优化

PPO 论文：https://arxiv.org/pdf/1707.06347
### 4.1.1 PPO四个主要模型

- Actor Model：演员模型，这就是我们想要训练的目标语言模型，类比考生。
- Reward Model：奖励模型，它的作用是计算实际收益，类比实考分数。
- Critic Model：评论家模型，它的作用是预期收益，类比老师订的目标分数。
- Reference Model：参考模型，它的作用是防止模型变动太大，类比 `KL(当前模型 || 固定的SFT模型)`

Actor与Reference的初始化模型就是SFT模型，Reward与Critic的初始化模型就是训练好的Reward模型，其中**Actor与Critic需要训练更新权重**，而**Reward与Reference是权重冻结**的。

基于人类偏好，训练一个 **Reward Model（奖励模型）**，它相当于变成一个“打分老师”，以后不用真人，每个回答都能自动给出一个“好不好”的分数。

接下来，用强化学习（通常是 PPO）去优化模型：**Actor Model（演员模型）** 不断生成回答，Reward Model 给它打分得到实际收益，同时 **Critic Model（评论家模型）** 提供预期表现，然后模型根据“预期表现和实际收益的差异”来调整自己，让高分回答更容易被生成。

在这个过程中，还会加上 **Clip（奖励截断）** 防止模型超常发挥带来巨额奖励，找到野路子从而学偏，以及用 **Reference 模型（就是 SFT 后的模型）做约束**，避免模型为了拿高分而脱离原本SFT训练好的旧模型太多。

| 模型   | Actor                             | Critic                          |
| ---- | --------------------------------- | ------------------------------- |
| 输入   | x                                 | x                               |
| 输出   | πθ(y \| x)                        | Vθ(x)                           |
| 标签   | 没有标签<br>Reward - Value好坏程度的加权学习信号 | 有标签，来自Reward模型的真实reward         |
| 简述   | 输入一段上下文x，输出下一个token y的概率分布        | 输入一段上下文x，输出模型按当前策略平均能得到多少 value |
| 训练目标 | 让“好的回答”概率变大<br>让“差的回答”概率变小        | 让预测的 value <br>接近真实 reward      |

在训练时会根据Actor和Critic的loss同时更新它俩的权重：
$$Loss=actor\_loss+0.1\times critic\_loss$$

### 4.1.2 PPO里Reward 奖励模型训练

这个模型的**底座就是第一阶段训练出的SFT模型，只是把最后一层改为一个神经元即可**，就是输出的分数，为一个回归模型，后续将用这个模型对每个回答进行打分评估。

奖励模型的核心是它的损失函数：
$$loss = -log(\sigma(r_\theta(x, y_c)-r_\theta(x,y_r))$$
其中c代表chosen，是排名较高的回答，r代表reject，是排名较低的回答。最终的目的就是使排名靠前的回答得分相应的变高。

```text
比如Q：苹果是什么？
A1：苹果是一种红色水果，可以润肺、解暑、开胃。
A2：苹果的品种繁多，根据颜色、大小、口感和用途等不同特点，可以分为多个品种。有的苹果品种适合鲜食，口感脆甜；有的适合烹饪，如做苹果派或苹果酱；还有的适合制酒或制醋。此外，美国苹果公司是全球知名的科技公司。
A3：苹果是水果。
```

人工标记员打分A2>A1>A3，我们把QA2&QA1、QA2&QA3、QA1&QA3两两放入模型进行打分，然根据损失函数对模型进行反向传播，最后就可以使A2的分数高于A1、A1的分数高于A3。根据不同答案的优先级顺序，就可以训练出奖励模型。

训练完毕后冻结参数以对不同的答案打分，奖励模型的好坏直接决定大模型最后的训练效果。


## 4.2 GRPO 组内相对策略优化

**PPO流程：**
![[Pasted image 20260410174450.png]]
输入问题q，经过策略模型(Policy Model)，生成回答o。接着输出o被输入到3个模型：
- 参考模型(Reference，冻结)：计算输出o与训练数据之间的KL散度，用于限制策略更新
- 奖励模型(Reward，冻结)：生成奖励r，是实际分数
- 价值模型(Value，更新)：输出o的预期分数
GAE优势 A=实际分数-预期分数。优势为在输出时生成了某个 token，如果这个token的优势大于0，那么我们应该增加生成该 token 的概率；反之，如果token的优势小于0，就减少生成该token的概率。最终，模型通过强化具有正优势的token，使得生成结果更加符合预期目标。


**GRPO流程：**
![[Pasted image 20260410174532.png]]
GRPO的核心就和它的名字一样，**创新点是使用组建关系去重塑优势的计算方法**：
1. 输入q，经过策略模型(Policy Model)，针对同一个q生成多个回答`o_1,o_2,...,o_G`
2. 接着输出o被输入到2个冻结的模型：
    - 参考模型(Reference，冻结)：计算输出o与训练数据之间的KL散度
    - 奖励模型(Reward，冻结)：生成每个回答的分数`r_1,r_2,...,r_G`
3. 计算这组回答分数的均值`\mu_{G}`和标准差`\sigma_{G}`
4. 计算每个样本的相对优势，然后强化优势使生成结果更加符合预期目标。

模型通过强化优势大于0的token生成，抑制优势小于0的token生成，使生成结果更加符合预期目标。

**GRPO更关注群体**：GPRO更关注群体表现，利用群体统计量计算相对优势，从而能够一次性提升整个团队（更多数据量）的表现，而PPO更注重单个样本。

**GRPO更省内存**：因为GPRO只利用Reward模型生成的分数计算损失，所以在GRPO的训练过程中，无需加载PPO训练过程中的Critic模型，能节省大量的 GPU 显存。

# 5. DPO 直接偏好优化

## 5.1 DPO

PPO训练流程繁琐，需要加载4个模型，所以占用显存很大，同时计算量也很大。

而DPO只有Actor和Reference两个模型，直接通过偏好数据进行微调，无需加载Reward和Critic这两个模型，效果简单粗暴，在使模型输出更符合人类偏好的同时，极大地缩短了训练时间和难度。

| **PPO：先造尺子，再用尺子量着学走路**                                                                                                                                                                                                                        | **DPO：直接看别人走的对错，调整步伐**                                                                                                                                                             |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1. PPO 损失函数需通过借助策略比值、优势函数及clip 裁剪操作限制策略更新幅度，追求长期回报。<br>    <br>2. PPO 损失函数的设计，主要就是为了使模型达到新旧模型输出分布合理与当前回答分高的双重目的。<br>    <br>3. PPO 需要 Reward + RL 两次训练，RL 阶段要加载4个模型并大量在线采样，耗时且耗显存。<br>    <br>4. PPO 的重点在于 reward 模型的质量，reward 模型的质量决定了模型的上限。 | 1. DPO 利用用户打标的 chosen 和 rejected 数据，最大化生成 chosen 数据的概率、减少生成 rejected 的概率。训练依赖用户排序或比较数据，更接近监督学习，从静态数据中训练，数据质量决定了模型最终的质量。<br><br>2. DPO 无需明确奖励函数，巧妙地将 Reward 和 RL 两个步骤合并，训练仅需加载2个模型。 |

DPO需要的数据与RLHF一致，都是经过人工排序后的QA语料对。不同的是DPO训练的数据格式核心是 “**输入 - 优选回答 - 次选回答**” 的三元组结构 ，其核心目的是让模型通过对比好回答（chosen）和差回答（rejected），学习人类的偏好倾向。

```Plaintext
{
  "input": "写一段关于“春天”的抒情文字，要求50字左右。",
  "chosen": "春风拂过，柳枝抽芽，樱花飘落如粉雪。田野里草色渐青，燕子衔泥筑巢，空气里满是湿润的花香，这是春天最温柔的呼吸。",
  "rejected": "春天来了，天气变暖了，花儿开了，草儿绿了，大家可以出去玩了。"
}
```

这种格式的核心作用是让 DPO 的损失函数（最大化的奖励模型对`chosen`数据和`rejected`数据的差值）有效优化模型，使其更倾向于生成`chosen`级别的回答：

$$  
\mathcal{L}_{\mathrm{DPO}}(\pi_\theta;\pi_{\mathrm{ref}})  
=  
-\mathbb{E}_{(x,y_c,y_r)\sim \mathcal{D}}  
\left[  
\log \sigma\left(  
\beta \log \frac{\pi_\theta(y_c\mid x)}{\pi_{\mathrm{ref}}(y_c\mid x)}  
-  
\beta \log \frac{\pi_\theta(y_r\mid x)}{\pi_{\mathrm{ref}}(y_r\mid x)}  
\right)  
\right].  
$$

yc是偏好数据对中好的回答 (chosen)，yr则是偏好数据对中坏的回答(rejected)

## 5.2 DAPO 提高裁剪上限&动态采样优化策略

论文地址：https://arxiv.org/pdf/2503.14476

DAPO 用 GRPO 的视角重新推导并改进偏好优化方法，从而连接 RL 和 DPO 两种范式。

1. 提高clip的上界
2. 动态采样（Dynamic Sampling）
3. Token-Level Gradient Loss
4. 软惩罚机制（Soft Punishment）
5. 移除 KL 散度
6. 在编程/数据问题中采用规则奖励

综上，DAPO 以 **Clip-Higher（突破低概率 token 涨幅限制）、动态采样（保障有效梯度）、Token-Level Gradient Loss（平衡长回答梯度权重）、软惩罚（约束回答长度）** 等方法精细化改造 GRPO，提升训练效率与稳定性。

# 7. 大模型推理策略

## 7.1 贪心策略

直接选择概率最高的词语，这种方法简单高效，但是可能会导致生成的文本过于单调和重复。

## 7.2 Beam Search 集束搜索策略

是对贪心策略一个改进。思路也很简单，就是稍微放宽一些考察的范围。在每一个时间步，不再只保留当前分数最高的1个输出，而是保留k个。当k=1时集束搜索就退化成了贪心搜索。

比如k=2: `I love` 下一个可能生成 `you(0.4), it(0.3), this(0.2)`，保留 `you` 和 `it` 这两个概率最大的，然后让`I love you` 和 `I love it` 分别再去生成下一个词，把所有候选排一起，选择概率最大的前k个。

## 7.3 Top-K

Top-K采样是对贪心策略的优化，在生成token的每一步，只从概率最高的 k 个单词中进行随机采样，而不考虑其他低概率的单词。

比如 `我喜欢` 后面有很多种选择`女孩(66.4%)，鞋子(19.9%)，大象(10.5%)，西瓜(3.2%)...`，在按概率排序的前k=3个选择里，按照剔除了后续单词重新计算的概率分布`女孩(72%)，鞋子(16%)，大象(11%)`进行选择

## 7.4 Top-P

Top-K有一个缺陷，那就是“K值取多少是最优的？”，这非常难确定。于是出现了动态设置 token 候选列表大小的策略，只留下累积概率高于指定阈值P的词，从而增加输出的多样性，同时避免低概率的词。

比如设置阈值=80%，那么留下按概率加总突破80%的选择`女孩(66.4%)+鞋子(19.9%)>80%`，按照重新计算的概率分布进行选择。

## 7.5 Temperature 温度

温度这个参数可以告诉机器如何在质量和多样性之间进行权衡：
- 较低的 Temperature意味着更高的质量。较低的Temperature 值 < 1（如 0.2 或 0.5），使模型更具信心，将大部分概率集中在最有可能的下一个词上，结果是输出更连贯且重复性更高，但创造力或探索性较低。
- 而较高的Temperature意味着更高的多样性。使模型的预测更加分散和“不确定”，让token的概率分布更加平滑并均匀，模型可以更频繁地从低概率的词中进行采样，从而生成更具创造性、探索性和“惊喜”的输出。

## 7.6 Frequency Penalty&Presence Penalty 频率惩罚，存在惩罚

频率惩罚和存在惩罚是另一种让模型在质量和多样性之间进行权衡的方法：
- 频率惩罚：让token每次在文本中出现都受到惩罚，进行累计。
- 存在惩罚：如果一个token已经在文本中出现过，就会受到固定惩罚。

增加输出内容的多样性，这使得对旧的和过度使用的token进行选择变得不太可能，从而让模型选择更新颖的token。

## 7.7 参数协同

一般是先使用Top-K选取前概率前50的token，然后使用Top-P保留概率的累计和达到0.7的单词，最后使用Temperature=0.7或1.5进行归一化。

在合理范围内调节参数：
	想要随机性：增大K，P，温度
	想要可靠性：减小K，P，温度
