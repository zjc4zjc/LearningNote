# LearningNote Image Protocol

本文档定义 `LearningNote` 仓库内 Markdown 文档的图片整理规范。后续整理图片时，统一按本协议执行。

## 1. 总体规则

- 如果某个 `md` 文件引用了图片，则在该 `md` 文件所在目录下创建对应的图片目录：`pics/<image_dir_name>`。
- 如果某个 `md` 文件没有引用任何图片，则不创建对应图片目录。
- 图片目录始终相对于当前 `md` 所在目录创建和引用，不跨一级目录，不复用其他笔记目录下的 `pics/` 内容。

## 2. 图片目录命名规则

- 记 `image_dir_name(md)` 为某个 `md` 文件对应的图片目录名。
- 默认情况下，`image_dir_name(md)` 等于该 `md` 文件去掉 `.md` 后的 basename。
- 如果 basename 命中章节命名规则，则使用章节缩写，而不是原 basename。

章节命名规则：

- 若 basename 以 `Chapter<number>` 或 `Ch<number>` 开头，则图片目录名统一写为 `chXX`。
- 其中 `XX` 为两位数字编号，不足两位时左侧补零。
- 例如：
  - `Chapter1 基础.md` -> `ch01`
  - `Ch1 基本概念.md` -> `ch01`
  - `Chapter2 Transformer.md` -> `ch02`
  - `Ch6 随机近似理论SA,SGD.md` -> `ch06`
  - `Chapter12 总结.md` -> `ch12`

非章节文件示例：

- `tut_环境变量的配置.md` -> `tut_环境变量的配置`
- `概述.md` -> `概述`

路径示例：

- `大模型/Chapter1 基础.md` 对应图片目录：`大模型/pics/ch01/`
- `CS常用操作/tut_环境变量的配置.md` 对应图片目录：`CS常用操作/pics/tut_环境变量的配置/`

## 3. 图片命名规则

- 图片按该 `md` 文件中出现的引用顺序依次命名。
- 命名格式为：`img-001`、`img-002`、`img-003`，依次类推。
- 保留原始文件扩展名，例如 `.png`、`.jpg`、`.svg`。

示例：

- `img-001.png`
- `img-002.png`
- `img-003.svg`

## 4. 图片存放规则

- 一个 `md` 文件所引用的图片，统一放在它自己的 `pics/<image_dir_name>` 文件夹中。
- 不再将该文件的图片散放在仓库根目录，或当前目录下的公共位置。
- 若旧图片已存在于公共目录，整理时应迁移到该 `md` 文件对应的图片目录中。
- 不允许让某个 `md` 文件直接引用其他 `md` 文件所属目录下的图片。

## 5. Markdown 引用规则

- 图片路径必须写为相对路径。
- 相对路径应以当前 `md` 文件所在位置为基准。
- 引用路径统一写为：`pics/<image_dir_name>/img-XXX.<ext>`

示例：

```md
![](pics/ch01/img-001.png)
![](pics/tut_环境变量的配置/img-001.png)
```

## 6. 整理时的执行顺序

每次整理某个 `md` 文件时，按以下步骤执行：

1. 扫描该文件中的全部图片引用。
2. 计算该文件对应的 `image_dir_name(md)`。
3. 按引用出现顺序重新编号。
4. 在当前 `md` 所在目录下的 `pics/` 中创建对应图片目录。
5. 将该文件使用的图片移动或整理到该目录中。
6. 将文档中的图片路径改为新的相对路径。
7. 检查所有图片引用都能正确指向实际文件。

## 7. 约束

- 不为没有图片引用的 `md` 文件创建空文件夹。
- 图片编号只在各自 `md` 文件内部独立计数，不跨文件共享编号。
- 后续新增图片时，继续按该文件内的出现顺序编号。
- 若对文档内容进行重排，导致图片顺序变化，则图片文件名和引用路径也应同步整理。
- 若发现两个不同目录下的 `md` 文件都存在 `ch01` 之类的缩写目录名，这是允许的；它们仍然各自归属于自己的父目录，不视为冲突。
